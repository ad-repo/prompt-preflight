# Security Policy

## Supported Versions

Security updates are currently applied to the latest `main` branch.

## Reporting a Vulnerability

Please do not post security vulnerabilities in public issues.

Instead:

1. Open a GitHub Security Advisory (preferred), or
2. Contact the maintainers directly through repository contact channels.

When reporting, include:

- Description of the vulnerability
- Impact and affected areas
- Reproduction steps or proof of concept
- Suggested mitigation (if known)

## Response Expectations

- Initial acknowledgment target: within 7 days
- Status updates: provided as investigation progresses
- Fix and disclosure timing: based on severity and user impact
